# HomeVital MVP v1 — 产品需求文档

> 本 PRD 已同步到当前开发主线。与旧 Phase 文档相比，MVP v1 现在以简化健康事实层、PydanticAI 应用内对话、成员档案模态框和每日 AI 摘要为核心。

## 产品定位

HomeVital 是一个私有部署的 AI 家庭健康管理助手。每个部署实例服务一个家庭，聚合全家健康信息，提供结构化健康档案、AI 对话、每日摘要和提醒能力。

## MVP v1 范围

### F1: 家庭成员管理

**概述：** 一个 HomeVital 实例 = 一个家庭空间。

**功能点：**

- F1.1 用户注册与登录
  - 第一个注册用户自动成为家庭管理员
  - 后续用户加入同一家庭空间
- F1.2 管理员添加成员
  - 支持无账号成员，例如老人和儿童
  - 成员后续可绑定账号
- F1.3 权限控制
  - 管理员可管理所有成员
  - 普通成员默认只能查看目录级基础信息
  - 健康数据访问按成员级授权控制

### F2: 健康档案

**概述：** 为每位家庭成员建立简化但统一的健康档案。

**功能点：**

- F2.1 基础档案
  - 创建成员时填写姓名、性别、出生日期、身高、血型等基础信息
- F2.2 健康数据录入
  - 手动录入核心指标：血压、血糖、体重、心率、血氧等
  - 记录睡眠、运动、就诊、用药、过敏和病史
- F2.3 对话驱动的结构化抽取
  - 用户通过 AI 对话提供症状、病史、处方、检查结果或附件上下文
  - AI 生成结构化草稿，用户确认后写入正式健康档案
  - 高风险写入必须经过确认
- F2.4 数据查看
  - 首页展示所有成员的健康摘要和当日提醒
  - 成员档案以多子页模态框查看健康数据、健康档案、就诊记录和药品管理
  - 支持关键指标趋势和历史列表

**MVP v1 当前资源范围：**

- `Observation`
- `SleepRecord`
- `WorkoutRecord`
- `Condition`
- `Medication`
- `Encounter`
- `HealthSummary`
- `CarePlan`

### F3: AI 功能

**概述：** AI 基于成员级权限和健康档案提供受控的对话式能力。

**功能点：**

- F3.1 对话式健康咨询
  - 支持自然语言问答
  - 支持文字、语音，以及作为对话上下文的附件输入
  - AI 只能在当前用户权限范围内读取和建议修改健康数据
- F3.2 结构化草稿确认
  - AI 可从对话中生成 Observation、Condition、Medication、Encounter 等草稿
  - 草稿以结构化卡片展示给用户确认
- F3.3 每日健康摘要与提醒
  - AI 每日为成员生成 `HealthSummary`
  - AI 每日刷新当日 `CarePlan`
  - 首页和成员概览展示这些结果

**MVP v1 当前不作为主路径的能力：**

- 独立文档资源管理
- 复杂多 Agent 工作流
- 深度医疗决策支持

### F4: MCP 服务接口（后续阶段）

**概述：** 将稳定的应用内能力对外暴露给 MCP 客户端。

**功能点：**

- F4.1 MCP Server
  - 暴露成员目录、健康数据读取和有限写入能力
  - 复用应用内服务层和权限校验

> F4 保留在产品范围内，但不是当前 Step 6-8 的实现主线。

### F5: 部署

**概述：** 保留自部署目标，同时承认当前开发主路径仍是本地 FastAPI + Vite。

**功能点：**

- F5.1 本地开发运行
  - 本地 SQLite
  - 分别启动前后端
- F5.2 目标部署骨架
  - `docker-compose.yml` 保留未来统一部署入口
  - 未来收敛到更完整的部署方式

## 非功能性需求

| 维度 | 要求 |
|---|---|
| 隐私 | 默认本地存储，外部调用仅限模型服务等必要依赖 |
| 安全 | 严格成员级权限控制，AI 不得绕过服务层 |
| 性能 | 面向家庭规模使用场景，无需高并发架构 |
| 可用性 | 本地部署可容忍短暂不可用，但数据必须可恢复 |
| 可扩展 | 保持未来设备接入、MCP、部署升级的扩展空间 |

## UI 参考

- 当前实现重构以 active plan 中的 16 页 Stitch 设计为准
- `stitch-screens/` 只保留为早期参考，不再作为当前 UI 目标

## 成功标准

1. 用户可注册、登录并管理家庭成员
2. 可为成员录入或确认结构化健康数据
3. 首页能展示成员健康摘要和当日提醒
4. AI 能在权限允许下进行问答、建议和草稿生成
5. 成员档案模态框能查看和编辑核心健康信息
6. 文档与实现不再继续围绕旧的 `DocumentReference` / 旧 orchestrator 路线扩展
