# HomeVital MVP v1 — 产品需求文档

## 产品定位

HomeVital 是一个私有部署的 AI 家庭健康管理助手。每个部署实例服务一个家庭，聚合全家健康信息，提供 AI 驱动的健康对话、文档结构化抽取和主动提醒。

## MVP v1 范围

MVP v1 聚焦以下核心能力，构建可用的端到端系统。

---

### F1: 家庭成员管理

**概述：** 一个 HomeVital 实例 = 一个家庭空间。

**功能点：**

- F1.1 用户注册与登录
  - 第一个注册用户自动成为家庭管理员
  - 后续用户注册后自动加入该家庭空间
- F1.2 管理员添加成员
  - 管理员可直接添加"无账号成员"（例如老人、儿童），其他用户可查询和管理其健康信息
  - 无账号成员后续可随时绑定账号
- F1.3 权限控制
  - 管理员：全部权限，可管理所有成员
  - 普通成员：可查看家庭成员目录中的基础信息；查看或修改其他成员的健康数据仍需授权
  - 访问控制粒度：成员级别（谁能看谁的信息、谁能改谁的信息）

**MVP v1 延后：**
- 管理员审核注册（后续版本考虑）

---

### F2: 健康档案

**概述：** 为每位家庭成员建立标准化的健康档案。

**功能点：**

- F2.1 基础档案
  - 创建成员时填写基础信息：姓名、性别、出生日期、血型、过敏史、既往病史
- F2.2 健康数据录入
  - 手动录入：血压、血糖、体重、体温、心率等常见指标
  - 记录就医信息：就诊日期、科室、诊断、用药
- F2.3 文档上传与结构化抽取
  - 支持上传 PDF 和图片（体检报告、检验单、处方单、出院小结）
  - AI 自动抽取关键信息，转为标准化健康事实
  - 用户确认/修正抽取结果后入库
- F2.4 数据查看
  - 家庭健康看板：总览所有成员状态
  - 成员详情页：时间线视图展示健康事件
  - 关键指标趋势图（血压、血糖等）

**MVP v1 延后：**
- 可穿戴设备数据同步（OpenWearable / HealthKit）
- 第三方 APP 数据接入

---

### F3: AI 功能

**概述：** 基于家庭健康数据提供 AI 驱动的交互能力。

**功能点：**

- F3.1 对话式健康咨询
  - 用户可以用自然语言查询家庭成员的健康信息
  - AI 可访问当前用户有权查看的健康数据作为上下文，并在权限允许时发起受控写入
  - 支持文字、语音输入，以及在对话入口上传图片/PDF 等附件
  - AI 对系统数据的读取与修改必须与当前用户权限模型保持一致
- F3.2 文档结构化抽取（与 F2.3 配合）
  - AI 解析上传的体检报告/检验单/处方单
  - 生成结构化的 FHIR 风格资源
- F3.3 主动健康提醒与定时任务
  - AI 根据成员健康数据生成每日事项通知
  - 支持用户自定义时间的提醒、随访或健康状态检查任务
  - 内容包括：用药提醒、复诊提醒、健康建议
  - 执行结果写入真实数据源，并展示在首页看板或成员档案页

**MVP v1 延后：**
- 深度健康趋势分析
- 多轮复杂推理

---

### F4: MCP 服务接口

**概述：** 将健康信息服务封装为 MCP Server，支持外部 AI 系统调用。

**功能点：**

- F4.1 MCP Server
  - 暴露家庭健康数据的查询能力（成员列表、健康记录、指标查询）
  - 暴露有限的写入能力（新增记录、更新提醒状态）
  - 遵循 MCP 协议规范
- F4.2 OpenClaw Skill（可选）
  - 封装为 OpenClaw Skill，支持 OpenClaw 调度
  - 通过 OpenClaw 实现主动推送（提醒、建议）

---

### F5: 部署

**概述：** 一键本地部署。

**功能点：**

- F5.1 Docker Compose 部署
  - 单个 `docker compose up` 启动所有服务
  - 包含所有必要组件（前端、后端、数据库、AI 服务）
  - 数据持久化到本地卷
- F5.2 初始化引导
  - 首次启动时引导创建管理员账号
  - 可选配置 AI 模型提供商（API Key 等）

---

## 非功能性需求

| 维度 | 要求 |
|------|------|
| 隐私 | 所有数据存储在本地，不外传至任何第三方（AI API 调用除外） |
| 安全 | 成员间数据隔离，按权限模型控制访问 |
| 性能 | 家庭规模（< 20 成员），无需考虑高并发 |
| 可用性 | 本地部署，容忍短暂不可用 |
| 可扩展 | 数据模型和 API 设计预留扩展空间（穿戴设备、更多数据源） |

## UI 参考

设计原型位于 `stitch-screens/`，包含三个核心页面：

1. **首页** (`screen-1/`) — 家人状态列表 + 今日提醒
2. **成员档案** (`screen-2/`) — 健康数据详情、指标趋势
3. **AI 对话** (`screen-3/`) — 浮层式 AI 助手对话界面

实现时以这些设计为参考方向，具体交互细节可调整。

---

## 成功标准

MVP v1 交付后应满足：

1. 可以通过 Docker Compose 一键启动
2. 管理员可以注册、添加家庭成员、管理权限
3. 可以为成员录入健康数据和上传体检报告
4. AI 可以基于健康数据进行对话，并在权限允许时驱动受控写入
5. 首页或成员档案页展示 AI 生成或按计划执行的健康提醒
6. MCP Server 可被外部工具调用
