# 家庭健康管理助手 — UI 设计说明

本目录包含 **家庭健康管理助手（KinCare）** 的 16 个 UI 设计页面，采用 Tailwind CSS 实现，风格统一、支持响应式布局。设计围绕「家庭健康守护」场景，覆盖登录、首页、AI 对话、成员管理、健康档案、就诊记录、药品管理等核心流程。

## 页面详细说明

### 01 — 首页 + AI 对话浮层（基础）
- **场景**：用户进入首页后，AI 助手以浮层形式出现，主动询问今日健康数据
- **要点**：背景模糊突出对话；Clarity Assistant 引导记录生理指标与心理情绪；用户可文字或语音输入
- **交互**：底部输入框 + 麦克风 + 发送；右上角关闭浮层

### 02 — 登录页
- **场景**：未登录用户访问应用时的入口
- **要点**：顶部品牌区、家庭运动氛围图、邮箱/密码表单、记住登录、忘记密码
- **交互**：立即登录、免费注册链接

### 03 — 首页（完整）
- **场景**：已登录用户的主工作台
- **要点**：左侧家人状态（张学礼、李秀兰、王建国），右侧今日提醒按时段分组（清晨叮嘱、午后守候）
- **交互**：刷新进度、底部健康输入框、麦克风、发送

### 04 — AI 对话 + 健康记录确认
- **场景**：用户输入「爸爸今天胃口不错…还测了空腹血糖 6.2」后，AI 抽取结构化数据
- **要点**：展示「正在识别健康记录项…」加载态；确认面板含血糖、心理情绪等可勾选项；支持编辑详情、忽略、确认保存
- **交互**：AI 建议是否记录饮食；底部「发现 2 条可录入数据」提示

### 05 — 成员档案弹窗（概览）
- **场景**：点击成员卡片后查看该成员完整档案
- **要点**：弹窗内左侧导航（概览、健康数据、健康档案、就诊记录、药品管理）；概览含基础信息、AI 健康摘要、今日提醒
- **交互**：编辑档案、关闭弹窗；今日提醒支持勾选完成

### 06 — 成员档案弹窗（编辑态）
- **场景**：在档案弹窗中编辑成员基础信息与提醒
- **要点**：基础信息可编辑（身高、体重、血型、腰围、过敏史）；今日提醒可勾选完成、删除、新建
- **交互**：完成按钮保存；AI 健康摘要区域置灰不可编辑

### 07 — 首页 + 用户下拉菜单
- **场景**：点击右上角头像展开账户与设置菜单
- **要点**：展示当前家庭管理员、管理家庭链接；成员管理、语言设置、退出登录
- **交互**：下拉菜单样式统一，支持 hover 高亮

### 08 — 成员管理弹窗
- **场景**：从用户菜单进入「成员管理」
- **要点**：成员列表展示头像、姓名、角色、账号绑定状态、权限标签（读取/写入/管理）
- **交互**：添加成员、关闭弹窗；每行右侧设置图标

### 09 — 成员健康数据
- **场景**：在成员档案内切换到「健康数据」标签
- **要点**：可折叠区块：慢病指标（血压、血糖）、生理指标（心率、血氧、体重、压力）、生活习惯（步数、活动消耗）、睡眠、运动记录
- **交互**：区块展开/收起；时间戳使用等宽字体

### 10 — 成员管理 + 权限设置
- **场景**：在成员管理弹窗中点击某成员的设置图标
- **要点**：展开该成员的权限详情；读取/写入/管理三项用 iOS 风格开关控制；账号所有者权限不可修改
- **交互**：开关切换；管理员权限全部开启且锁定

### 11 — 成员管理 + 添加成员表单
- **场景**：在成员管理弹窗中点击「添加成员」
- **要点**：表单含头像上传、成员姓名、性别选择、出生日期
- **交互**：取消、确认添加；头像为可选上传

### 12 — 成员健康档案
- **场景**：在成员档案内切换到「健康档案」标签
- **要点**：现病（原发性高血压、2 型糖尿病）、既往病史（急性肺炎、左桡骨骨折）、家族病史（冠心病、脑卒中）、过敏与禁忌（青霉素、磺胺类）
- **交互**：可折叠区块；过敏项使用警告样式突出

### 13 — 成员健康档案（编辑态）
- **场景**：在健康档案页进入编辑模式
- **要点**：现病、既往病史、家族病史、过敏与禁忌均可增删改；每条支持编辑、删除、排序
- **交互**：添加条目、保存更改；家族病史新增时展示虚线框表单

### 14 — 成员就诊记录
- **场景**：在成员档案内切换到「就诊记录」标签
- **要点**：历史就诊列表，每条含日期、机构、科室、类型（复查/初诊/检查）、诊断摘要、接诊医生、详细诊断、附件
- **交互**：可展开查看详情；按日期排序、筛选类型；加载更多

### 15 — 成员就诊记录（编辑态）
- **场景**：在就诊记录页进入编辑模式
- **要点**：新增就诊记录表单（日期、机构、科室、医生、诊断摘要、详细诊断、附件上传）；已有记录支持删除
- **交互**：编辑模式标识；保存记录、取消

### 16 — 成员药品管理
- **场景**：在成员档案内切换到「药品管理」标签
- **要点**：正在服用（阿司匹林、氨氯地平等，含规格、用途、用法、开始时间）；已停用（感冒灵、布洛芬等，含停止日期）
- **交互**：查看更多历史记录；服用中/已停用视觉区分

## 设计规范

- **色彩**：暖奶油背景（`--warm-cream`）、柔和蓝（`--gentle-blue`）、森林绿（`--forest-green`）、暖灰（`--warm-gray`）
- **字体**：Quicksand / Inter + Noto Sans SC，时间等使用 JetBrains Mono
- **图标**：Material Symbols Outlined
- **圆角**：卡片 `rounded-[2rem]` / `rounded-[2.5rem]`，按钮 `rounded-full` / `rounded-xl`
- **阴影**：`apple-shadow`、`enterprise-shadow`、`sheet-shadow` 等轻量阴影

## 使用说明

每个页面的 `code.html` 为独立 HTML 文件，可直接在浏览器中打开预览。依赖 Tailwind CDN 与 Google Fonts，需联网加载。
